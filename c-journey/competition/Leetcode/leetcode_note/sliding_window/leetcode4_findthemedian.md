# 4. 寻找两个正序数组的中位数

**标签**：`二分查找` `数组`  
**难度**：困难  
**日期**：2026-05-10

---

## 思路

- 第一感：合并后排序再找中位数，O(m+n)，但题目要求 O(log(m+n))
- log一般就涉及到二分之类的思想
- 优化：有序数组 + log 复杂度 → 想到二分
- 关键：**不是找中位数值，而是找一个切割点**，把两个数组都切成左右两半，满足：
  1. 左边元素总数 = 右边（或左边多 1 个）
  2. 左边所有元素 ≤ 右边所有元素

-   比较逻辑：
-   主要是先从小数组开始，小数组左半部分的最大值要小于大数组右半部分的最小值
-   小数组右半部分的最小值大于大数组左半部分的最大值，这样才能确定左边元素全部小于右边
