## 1. 考研核心考点：算法复杂度对比

本题的核心是考查：**如何同时兼顾“频繁的位置对调”与“高频的关键字查询”？**

| 方案 | 交换操作复杂度 | 查询操作复杂度 | 总时间复杂度 | 408 考点评价 |
| :--- | :---: | :---: | :---: | :--- |
| **方案一：乱序静态数组**<br>(原题解烂代码) | $O(1)$ | $O(N)$ | $O(T + M \times N)$ | **不及格**。逻辑链路自我矛盾，高频查询直接退化为线性碰撞，5万数据量下必超时（TLE）。 |
| **方案二：人类直觉流**<br>(整体结构体交换) | $O(1)$ | $O(N)$ | $O(T + M \times N)$ | **及格**。符合物理直觉，直接 swap 结构体。代码干净，但未解决乱序后的查询高复杂度问题。 |
| **方案三：408 满分正解**<br>(二分降维 + 双向映射) | **$O(1)$** | **$O(\log N)$** | **$O((N+T+M)\log N)$** | **满分**。核心思想是**降维打击**。初始化时将“字符串”彻底剥离为“整数 ID”，后续全部转为 $O(1)$ 的整型数组操作。 |

## 2. 算法核心架构图（垂直流向）

```text
【阶段一：初始化定义】
   输入 N 个字符串名字 ➔ 分配唯一的整数 ID (1 ~ N)
         │
         ▼
【阶段二：建立二分索引】
   对 [名字-ID 结构体] 按名字字典序进行快速排序 (qsort)
   目的：后续可通过二分查找实现：字符串 ➔ 整数 ID (解耦关键)
         │
         ▼
【阶段三：魔术交换 (O(1) 核心)】
   读入杯子位置 p1, p2
   1. 杯子视角对调：swap(cup_to_toy[p1], cup_to_toy[p2])
   2. 玩具视角更新：toy_to_pos[toy_id] = 最新位置
         │
         ▼
【阶段四：极速查询】
   输入待查名字 ➔ 二分查找 O(log N) 转换出 ID ➔ O(1) 直接读取 toy_to_pos[ID]

```

## 3. 核心知识点：双向映射表机制（间接寻址）

在交换阶段，为了彻底干掉慢如牛爬的字符串对比（`strcmp`），引入两个**整型数组**互为逆映射：

* **`cup_to_toy[位置] = 玩具ID`** ：实现 **位置 ➔ 玩具** 的映射（交换时用作定位雷达）。
* **`toy_to_pos[玩具ID] = 位置`** ：实现 **玩具 ➔ 位置** 的映射（查询时直接一步到位）。

## 4. 408 高频考点：核心代码片段

### ① 降维关键：二分查找（字符串 ➔ 玩具 ID）

```c
int find_toy_id(char* name, int N) {
    int low = 1, high = N;
    while (low <= high) {
        int mid = low + (high - low) / 2; // 408 防溢出标准写法
        int cmp = strcmp(identity[mid].name, name);
        if (cmp == 0) return identity[mid].id; // 命中，返回固定数字 ID
        else if (cmp < 0) low = mid + 1;       // 目标在右半区
        else high = mid - 1;                  // 目标在左半区
    }
    return -1;
}

```

### ② 核心对调：$O(1)$ 数组双向映射更新

```c
// 读入要交换的两个杯子位置 p1, p2
int toy1 = cup_to_toy[p1]; // 探测：当前 p1 杯子里是哪个玩具 ID
int toy2 = cup_to_toy[p2]; // 探测：当前 p2 杯子里是哪个玩具 ID

// 1. 杯子视角：物理对调（两个杯子里的玩具 ID 互换）
cup_to_toy[p1] = toy2;
cup_to_toy[p2] = toy1;

// 2. 玩具视角：账本更新（两个玩具记住自己最新的杯子位置）
toy_to_pos[toy1] = p2;
toy_to_pos[toy2] = p1;

```

### ③ 极速查询：$O(1)$ 账本直出

```c
// 1. 先用二分查找拿到玩具的固定 ID (O(log N))
int toy_id = find_toy_id(query_name, N);

// 2. 通过 ID 直接查表，瞬间秒杀输出 (O(1))
printf("%d\n", toy_to_pos[toy_id]);

```

## 💡 408 考研老学长避坑心得

1. **只要涉及高频的“字符串查询”，首选散列（Hash）或排序后二分。** 绝对不能在循环内用 `strcmp` 进行线性遍历！
2. **处理复杂联动关系时，引入“间接寻址”（双向映射）。** 巧妙利用 `A[i] = b` 和 `B[b] = i`，可以在元素频繁移动时依然保持 $O(1)$ 的追踪效率。
