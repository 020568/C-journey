# MT2018 找1 —— DP思路的本质理解

## 一、用户的直觉（完全正确）

> 从 `111` 到 `1111`，加上长度 `4` 就相当于是多加了：`1111`、`111`、`11`、`1`。  
> 也就是**末尾多加一个 `1` 带来的新增子串**。

这个直觉就是 DP 的核心，下面把它形式化。

---

## 二、DP 定义

设 `dp[i]` = **以第 `i` 个字符结尾的、全为 `1` 的子串个数**。

### 为什么这样定义？
因为题目要求"所有字符都为1的子串的个数"，我们可以按**子串的结尾位置**进行分类计数，最后把每一类的个数加起来就是答案。

---

## 三、递推关系（用户的直觉的数学表达）

### 情况 1：`s[i] == '0'`
以 `0` 结尾不可能形成全1子串：
```
dp[i] = 0
```

### 情况 2：`s[i] == '1'`
这是关键。假设前面 `s[i-1]` 结尾处有 `dp[i-1]` 个全1子串：

```
...111      (以 i-1 结尾的 dp[i-1] 个全1子串)
     ↑
     i-1
```

现在在末尾多加一个 `1`（位置 `i`）：
```
...1111
     ↑
     i
```

**之前那 `dp[i-1]` 个子串，每个往后延长一位**，都变成了以 `i` 结尾的新全1子串：
- 原来 `"1"`（结尾在 `i-1`）→ 变成 `"11"`（结尾在 `i`）
- 原来 `"11"`（结尾在 `i-1`）→ 变成 `"111"`（结尾在 `i`）
- ...
- 原来 `"111"`（结尾在 `i-1`）→ 变成 `"1111"`（结尾在 `i`）

**新增了 `dp[i-1]` 个**

另外，**单独一个 `s[i]` 自己**也构成一个长度为1的全1子串：
- `"1"`（只包含 `s[i]`）

**新增了 1 个**

所以：
```
dp[i] = dp[i-1] + 1
```

---

## 四、用户的直觉验证

以 `111` → `1111` 为例：

| 阶段 | 字符串 | dp 值 | 以当前位结尾的全1子串 |
|------|--------|-------|----------------------|
| i=0 | `1` | `1` | `"1"` |
| i=1 | `11` | `2` | `"1"(s[1]), "11"` |
| i=2 | `111` | `3` | `"1", "11", "111"` |
| i=3 | `1111` | `4` | `"1", "11", "111", "1111"` |

从 `111`（`dp[2]=3`）到 `1111`（`dp[3]=4`）：
- 延长了之前的 3 个：`"1"→"11"`, `"11"→"111"`, `"111"→"1111"`
- 新增了单独的 1 个：`"1"(s[3])`
- **总共新增 4 个**，恰好是 `dp[3] = 4`

而答案 `ans` 就是累加所有 `dp[i]`：
```
ans = dp[0] + dp[1] + dp[2] + dp[3] = 1 + 2 + 3 + 4 = 10
```

用分块法验证：`k=4`，`4*(4+1)/2 = 10` ✅

---

## 五、两种视角的统一

| 视角 | 计算方式 | 本质 |
|------|---------|------|
| **分块法** | 一段连续 `k` 个1，贡献 `k*(k+1)/2` | 等差数列求和 |
| **DP法** | 逐个位置累加 `dp[i]`，每个 `dp[i]` 是连续1的长度 | 把三角数拆成 `1+2+3+...+k` |

两者数学等价：
```
1 + 2 + 3 + ... + k = k*(k+1)/2
```

DP 法的好处是**不需要先分段**，边遍历边算，代码更简洁。

---

## 六、滚动数组优化

观察递推式 `dp[i] = dp[i-1] + 1`，发现只依赖前一个状态，所以不需要开数组：

```c
long long cur = 0;  // 滚动变量，代替 dp[i]

for (int i = 0; s[i] != '\0'; i++) {
    if (s[i] == '1') {
        cur++;          // cur = dp[i-1] + 1
    } else {
        cur = 0;        // dp[i] = 0
    }
    ans = (ans + cur) % MOD;  // 实时累加
}
```

`cur` 就是当前的 `dp[i]`，同时代表了"以当前位结尾的全1子串个数"，也就是"当前连续1的长度"。

---

## 七、复杂度

| 指标 | 值 | 说明 |
|------|-----|------|
| **时间** | O(n) | 遍历字符串一次 |
| **空间** | O(1) | 只用 `cur` 和 `ans` 两个变量 |

---

## 八、一句话总结

> **DP 的本质：每遇到一个新的 `1`，它可以把之前所有以相邻位结尾的全1子串"延长"，同时还能单独成一个新子串。所以新增个数 = 之前的个数 + 1。**
> 
> 用户的直觉——"加上长度4就相当于多加了1111、111、11、1"——完全抓住了这个递推的核心。
