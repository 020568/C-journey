## 一、算法思想 [⬆️ 返回目录](#toc)

```
┌─────────────────────────────────────────────────────────────┐
│                    Dijkstra 核心思想                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   贪心策略：每次选距离源点「最近」的未确定顶点                │
│                                                             │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐              │
│   │ 已确定  │───▶│ 候选集  │───▶│ 选最小  │              │
│   │ 集合 S  │    │  未确定  │    │  加入S  │              │
│   └─────────┘    └─────────┘    └─────────┘              │
│                                                             │
│   关键操作：松弛 (Relaxation)                               │
│   if dist[v] > dist[u] + w(u,v):                           │
│       dist[v] = dist[u] + w(u,v)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 适用条件

| 条件 | 说明 | ⚠️ 注意 |
|------|------|---------|
| **单源** | 从一个起点到所有其他点 | 不是多源 |
| **非负权** | 所有边权 ≥ 0 | **不能处理负权边！** |
| **有向/无向** | 两者均可 | 无向图可看作双向有向 |

### 1.2 核心变量

| 变量 | 含义 | 初始值 |
|------|------|--------|
| `dist[]` | 源点到各顶点的最短距离 | `dist[s]=0`, 其余=∞ |
| `visited[]` | 标记顶点是否已确定 | 全 false |
| `path[]` | 记录前驱顶点（可选，用于还原路径） | 全 -1 |

## 二、算法步骤 [⬆️ 返回目录](#toc)

### 2.1 标准流程

```
Step 1: 初始化
    dist[s] = 0
    dist[其他] = ∞
    visited[全] = false

Step 2: 循环 n 次
    for i = 1 to n:
        ① 在未确定顶点中，选 dist 最小的顶点 u
        ② 标记 visited[u] = true（u 已确定）
        ③ 对 u 的每个邻接点 v：
            if !visited[v] 且 dist[v] > dist[u] + w(u,v):
                dist[v] = dist[u] + w(u,v)   // 松弛
                path[v] = u                    // 记录前驱

Step 3: 输出 dist[]（源点到各点的最短距离）
```

### 2.2 流程图

```
        开始
          │
          ▼
    ┌─────────────┐
    │  初始化     │
    │ dist[s]=0   │
    │ 其余=∞      │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐     是    ┌─────────────┐
    │ 还有未确定  │────────▶│ 选dist最小   │
    │  顶点？     │          │  的顶点u     │
    └──────┬──────┘          └──────┬──────┘
           │否                      │
           ▼                        ▼
    ┌─────────────┐          ┌─────────────┐
    │   结束      │          │ visited[u]=T│

                                    ▼
                           ┌─────────────┐
                           │ 遍历u的邻接点 │
                           │   执行松弛    │

                                  │
                                  └──────────────┐
                                                 │
                                          ┌──────┴──────┐
                                          │ 回到「还有  │
                                          │ 未确定顶点？」│
                                          └─────────────┘
```

## 三、手写推演示例 [⬆️ 返回目录](#toc)

### 3.1 示例图

```
              4
        ① ────────> ②
        │    2      │
      1 │           │ 1
        ▼           ▼
        ③ ────────→ ④
              2

顶点：1, 2, 3, 4
边权：(1,2)=4, (1,3)=1, (3,2)=2, (3,4)=2, (2,4)=1
源点：1
```

### 3.2 推演表格

| 轮次 | 选中的u | dist[1] | dist[2] | dist[3] | dist[4] | 已确定集合S |
|:----:|:-------:|:-------:|:-------:|:-------:|:-------:|:-----------:|
| 初始 | — | **0** | ∞ | ∞ | ∞ | {1} |
| 第1轮 | 1 | **0** | 4 | 1 | ∞ | {1} |
| 第2轮 | 3 | **0** | 3 | **1** | 3 | {1,3} |
| 第3轮 | 2 | **0** | **3** | **1** | 3 | {1,3,2} |
| 第4轮 | 4 | **0** | **3** | **1** | **3** | {1,3,2,4} |

### 3.3 推演过程详解

```
第1轮：选中顶点 1（源点）
    松弛边 (1,2): dist[2] = min(∞, 0+4) = 4
    松弛边 (1,3): dist[3] = min(∞, 0+1) = 1

第2轮：选中顶点 3（dist最小=1）
    松弛边 (3,2): dist[2] = min(4, 1+2) = 3  ✓ 更新！
    松弛边 (3,4): dist[4] = min(∞, 1+2) = 3

第3轮：选中顶点 2（dist=3，与4并列但先选2）
    松弛边 (2,4): dist[4] = min(3, 3+1) = 3  ✗ 不更新

第4轮：选中顶点 4（dist=3）
    无邻接点需要松弛

最终结果：
    1→2 最短距离 = 3（路径：1→3→2）
    1→3 最短距离 = 1（路径：1→3）
    1→4 最短距离 = 3（路径：1→3→4）
```

## 四、代码实现 [⬆️ 返回目录](#toc)

### 4.1 邻接矩阵版（适合稠密图）

```c
#define MAXV 100
#define INF 0x3f3f3f3f

// g[u][v] = 边权，不存在则为INF
void Dijkstra(int g[MAXV][MAXV], int n, int s, int dist[], int path[]) {
    bool visited[MAXV] = {false};

    // 初始化
    for (int i = 0; i < n; i++) {
        dist[i] = g[s][i];
        path[i] = (dist[i] < INF && i != s) ? s : -1;
    }
    dist[s] = 0;
    visited[s] = true;

    // 进行 n-1 轮
    for (int i = 1; i < n; i++) {
        // ① 找未确定的最小dist
        int u = -1, minDist = INF;
        for (int j = 0; j < n; j++) {
            if (!visited[j] && dist[j] < minDist) {
                minDist = dist[j];
                u = j;
            }
        }

        if (u == -1) break;  // 剩余顶点不可达
        visited[u] = true;

        // ② 松弛u的所有邻接点
        for (int v = 0; v < n; v++) {
            if (!visited[v] && g[u][v] < INF) {
                if (dist[v] > dist[u] + g[u][v]) {
                    dist[v] = dist[u] + g[u][v];
                    path[v] = u;  // 记录前驱
                }
            }
        }
    }
}
```

### 4.2 邻接表版 + 堆优化（适合稀疏图）

```c
// 邻接表节点
typedef struct Edge {
    int v;      // 邻接点
    int w;      // 边权
    struct Edge *next;
} Edge;

// 小顶堆节点
typedef struct {
    int dist;   // 距离
    int v;      // 顶点编号
} HeapNode;

// 堆优化Dijkstra，时间复杂度 O((V+E)logV)
void Dijkstra_Heap(Edge *adj[], int n, int s, int dist[], int path[]) {
    bool visited[MAXV] = {false};
    HeapNode heap[MAXV];
    int heapSize = 0;

    // 初始化
    for (int i = 0; i < n; i++) dist[i] = INF;
    dist[s] = 0;

    // 源点入堆
    heap[heapSize++] = (HeapNode){0, s};

    while (heapSize > 0) {
        // 弹出堆顶（dist最小的）
        HeapNode top = heap[0];
        heap[0] = heap[--heapSize];
        // ... 向下调整堆 ...

        int u = top.v;
        if (visited[u]) continue;  // 已确定则跳过
        visited[u] = true;

        // 松弛邻接点
        for (Edge *e = adj[u]; e != NULL; e = e->next) {
            int v = e->v, w = e->w;
            if (!visited[v] && dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                path[v] = u;
                // v入堆
                heap[heapSize++] = (HeapNode){dist[v], v};
                // ... 向上调整堆 ...
            }
        }
    }
}
```

### 4.3 路径还原（利用path数组）

```c
// 从终点倒推路径
void PrintPath(int path[], int s, int t) {
    if (t == s) {
        printf("%d", s);
```

## 五、正确性证明（理解） [⬆️ 返回目录](#toc)

### 5.1 核心结论

> **当顶点 u 被标记为「已确定」时，dist[u] 就是源点到 u 的最短距离。**

### 5.2 证明思路（反证法）

```
假设：u 是被选中的顶点，但存在另一条更短路径 s→...→u

分析：
    这条更短路径上，必然存在一个「未确定」顶点 x（因为 u 刚被确定）
    dist[x] ≤ 这条路径长度 < dist[u]

    但算法选的是 dist 最小的未确定顶点，
    如果 dist[x] < dist[u]，算法应该先选 x 而不是 u。

    矛盾！因此假设不成立。

结论：dist[u] 确实是最短距离。
```

### 5.3 为什么不能有负权边？

```
反例：
    s ──(5)──▶ a ──(-10)──▶ b
    s ──(1)──▶ c ──(1)──▶ b

    Dijkstra过程：
    第1轮：选 a（dist=5），确定 a
           松弛 a→b: dist[b] = 5 + (-10) = -5
    第2轮：选 c（dist=1），确定 c
           松弛 c→b: dist[b] = min(-5, 1+1) = -5

    看起来没问题？但如果图更复杂：

    s ──(3)──▶ a ──(-2)──▶ b ──(1)──▶ c
    s ──(1)──▶ d ──(2)──▶ c

    第1轮：选 d（dist=1），确定 d
    第2轮：选 a（dist=3），确定 a
           松弛 a→b: dist[b] = 3-2 = 1
           松弛 b→c: dist[c] = 1+1 = 2
    第3轮：选 b（dist=1），确定 b
    第4轮：选 c（dist=2），确定 c

    但实际最短路径：s→a→b→c = 3-2+1 = 2 ✓（这个例子碰巧对了）

    更典型的反例：
    s ──(2)──▶ a ──(3)──▶ b
    s ──(4)──▶ c ──(-3)──▶ a

    第1轮：选 a（dist=2），确定 a
           松弛 a→b: dist[b] = 5
    第2轮：选 c（dist=4），确定 c
           松弛 c→a: dist[a] = min(2, 4-3) = 1  ✗ a已确定，无法更新！

    实际最短路径：s→c→a = 1，但 a 已经被错误地确定为 2 了！
```

## 六、时间复杂度分析 [⬆️ 返回目录](#toc)

| 实现方式 | 时间复杂度 | 适用场景 |
|:---------|:----------:|:---------|
| 邻接矩阵 + 线性查找 | **O(V²)** | 稠密图（E≈V²） |
| 邻接表 + 二叉堆 | **O((V+E)logV)** | 稀疏图 |
| 邻接表 + 斐波那契堆 | **O(VlogV + E)** | 理论最优（不常考） |

### 6.1 O(V²) 推导

```
外层循环 V 轮
    每轮线性查找最小 dist：O(V)
    每轮松弛所有邻接点：O(V) 邻接矩阵 / O(degree) 邻接表

总复杂度：V × (V + V) = O(V²)  邻接矩阵
          V × (V + E/V) ≈ O(V²) 稠密图
```

### 6.2 O((V+E)logV) 推导

```
每个顶点最多入堆一次：V 次入堆/出堆，每次 O(logV) → O(VlogV)
每条边最多引发一次松弛：E 次可能的入堆，每次 O(logV) → O(ElogV)

总复杂度：O(VlogV + ElogV) = O((V+E)logV)
```

## 七、易错点 & 408陷阱 [⬆️ 返回目录](#toc)

### 7.1 常考陷阱速查

| 陷阱 | 说明 | 应对 |
|------|------|------|
| **负权边** | Dijkstra不能处理，会出错 | 看到负权立刻换 Bellman-Ford / SPFA |
| **相等距离** | 多个顶点dist相同时，选哪个？ | 任选一个，结果都正确 |
| **不可达顶点** | 某些顶点从源点无法到达 | dist保持∞，path保持-1 |
| **自环/重边** | 图中可能有自环或多条同向边 | 取最小边权存入 |
| **起点终点相同** | dist[s] 始终为 0 | 注意初始化别搞错 |
| **邻接矩阵INF** | 初始化时别用0表示无边 | 用极大值（如0x3f3f3f3f） |

### 7.2 408真题常见问法

```
Q1: "写出Dijkstra算法求从顶点X到各顶点的最短路径的过程"
    → 按轮次写表格（顶点、dist数组、已确定集合）

Q2: "上述过程的时间复杂度为？"
    → 看存储结构：邻接矩阵O(V²)，邻接表+堆O((V+E)logV)

Q3: "图中存在负权边，能否用Dijkstra？"
    → 不能。反例 + 推荐Bellman-Ford

Q4: "Dijkstra属于哪种算法策略？"
    → 贪心算法（不是动态规划！）
```

## 八、对比总结 [⬆️ 返回目录](#toc)

### 8.1 最短路径算法全家桶

```
┌─────────────────────────────────────────────────────────────┐
│                      最短路径算法对比                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Dijkstra      │  单源  │  非负权  │  贪心   │ O(V²)/O(ElogV) │
│   ─────────────────────────────────────────────────────────  │
│   Bellman-Ford  │  单源  │  可有负权│  DP     │ O(VE)          │

│                                                             │

```

### 8.2 Dijkstra vs Prim（容易混淆！）

| 对比项 | Dijkstra | Prim |
|--------|----------|------|
| **目标** | 最短路径（距离最小） | 最小生成树（权值和最小） |
| **dist含义** | 源点到该点的距离 | 该点到生成树的最小边权 |
| **更新方式** | `dist[v] = dist[u] + w` | `dist[v] = w(u,v)` |
| **结果** | 一棵最短路径树 | 一棵最小生成树 |
| **共同点** | 都是贪心，都是选最小dist | — |

### 8.3 一句话记忆

> **Dijkstra =  Prim的dist加上「累积」**
>
> Prim：`dist[v] = 边(u,v)的权值`（只关心连接到树的最小边）
>
> Dijkstra：`dist[v] = dist[u] + 边(u,v)的权值`（关心从源点出发的总距离）
