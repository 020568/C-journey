## 一、图的基本概念 [⬆️ 返回目录](#toc)

### 1.1 定义与术语

| 概念 | 要点 | 408常考 |
|------|------|---------|
| **有向图** | 边有方向，`<u,v>` 与 `<v,u>` 不同 | ⭐⭐⭐ |
| **无向图** | 边无方向，`(u,v)` = `(v,u)` | ⭐⭐⭐ |
| **度** | 无向图：度数=边数×2；有向图：入度+出度=边数 | ⭐⭐⭐ |
| **路径/回路** | 简单路径：顶点不重复；简单回路：除首尾外不重复 | ⭐⭐ |
| **连通性** | 无向：连通图/连通分量；有向：强连通/弱连通 | ⭐⭐⭐ |
| **完全图** | 无向：n(n-1)/2 条边；有向：n(n-1) 条边 | ⭐⭐ |
| **稠密/稀疏** | 边数接近完全图为稠密，反之为稀疏 | ⭐ |

### 1.2 有向图 vs 无向图 对比

```
┌─────────────────┬─────────────────┐
│    无向图       │    有向图       │
├─────────────────┼─────────────────┤
│ 边 (u,v)        │ 弧 <u,v>        │
│ 无方向          │ u→v 有方向      │
│ 度 = 邻接边数   │ 入度+出度       │
│ 连通分量        │ 强连通分量      │
│ 生成树          │ 有向树/有向无环 │
└─────────────────┴─────────────────┘
```

## 二、图的存储结构 [⬆️ 返回目录](#toc)

### 2.1 邻接矩阵

```
        v0  v1  v2  v3
    v0 [ 0   1   1   0 ]
    v1 [ 1   0   1   1 ]
    v2 [ 1   1   0   0 ]
    v3 [ 0   1   0   0 ]
```

| 特性 | 说明 |
|------|------|
| 空间复杂度 | O(n²) |
| 适合场景 | 稠密图 |
| 查边效率 | O(1) |
| 求度效率 | O(n) |

### 2.2 邻接表

```
    顶点数组          边链表
    ┌─────┐
    │ v0  │───▶ v1 ──▶ v2 ──▶ NULL
    ├─────┤
    │ v1  │───▶ v0 ──▶ v2 ──▶ v3 ──▶ NULL
    ├─────┤
    │ v2  │───▶ v0 ──▶ v1 ──▶ NULL
    ├─────┤
    │ v3  │───▶ v1 ──▶ NULL
    └─────┘
```

| 特性 | 说明 |
|------|------|
| 空间复杂度 | O(n+e) |
| 适合场景 | 稀疏图 |
| 查边效率 | O(degree) |
| 求度效率 | O(1) 或 O(degree) |

### 2.3 十字链表 / 邻接多重表（了解）

- **十字链表**：有向图的优化存储
- **邻接多重表**：无向图的优化存储

## 三、图的遍历 [⬆️ 返回目录](#toc)

### 3.1 DFS（深度优先搜索）

```
伪代码：
    visited[v] = true
    for w in adj(v):
        if !visited[w]:
            DFS(w)
```

| 要点 | 内容 |
|------|------|
| 辅助结构 | visited数组 + 递归栈 |
| 时间复杂度 | O(n+e) 邻接表 / O(n²) 邻接矩阵 |
| 空间复杂度 | O(n) |
| 应用 | 连通分量、拓扑排序、判断环 |

### 3.2 BFS（广度优先搜索）

```
伪代码：
    visited[v] = true
    Queue.push(v)
    while !Queue.empty():
        u = Queue.pop()
        for w in adj(u):
            if !visited[w]:
                visited[w] = true
                Queue.push(w)
```

| 要点 | 内容 |
|------|------|
| 辅助结构 | visited数组 + 队列 |
| 时间复杂度 | O(n+e) 邻接表 / O(n²) 邻接矩阵 |
| 空间复杂度 | O(n) |
| 应用 | 最短路径（无权图）、层次遍历 |

### 3.3 DFS vs BFS 对比

```
┌─────────────────┬─────────────────┐
│      DFS        │      BFS        │
├─────────────────┼─────────────────┤
│ 栈/递归         │ 队列            │
│ 一条路走到底    │ 一层一层扩展    │
│ 适合找连通块    │ 适合找最短路径  │
│ 可能栈溢出      │ 内存消耗较大    │
│ 拓扑排序        │ 无权图最短路径  │
└─────────────────┴─────────────────┘
```

## 四、最小生成树 [⬆️ 返回目录](#toc)

### 4.1 基本概念

- **生成树**：包含图中所有顶点的极小连通子图（n个顶点，n-1条边）
- **最小生成树（MST）**：权值之和最小的生成树
- **性质**：MST不唯一，但权值和唯一

### 4.2 Prim算法

```
思路：从某个顶点出发，每次选连接树与非树顶点的最小边

伪代码：
    U = {v0}
    while |U| < n:
        选 (u,v) 满足 u∈U, v∉U 且权值最小
        将 v 加入 U，边加入 MST
```

| 要点 | 内容 |
|------|------|
| 时间复杂度 | O(n²) 邻接矩阵 / O((n+e)logn) 堆优化 |
| 适合场景 | 稠密图 |
| 核心思想 | **加点法** |

### 4.3 Kruskal算法

```
思路：按边权从小到大选边，不形成环则加入

伪代码：
    所有边按权值排序
    for each edge (u,v) in sorted order:
        if Find(u) != Find(v):
            Union(u,v)
            加入 MST
```

| 要点 | 内容 |
|------|------|
| 时间复杂度 | O(eloge) 排序 + O(e·α(n)) 并查集 |
| 适合场景 | 稀疏图 |
| 核心思想 | **加边法** + 并查集判环 |

### 4.4 Prim vs Kruskal 对比

```
┌─────────────────┬─────────────────┐
│     Prim        │    Kruskal      │
├─────────────────┼─────────────────┤
│ 加点法          │ 加边法          │
│ 维护一个集合U   │ 维护并查集      │
│ O(n²)           │ O(eloge)        │
│ 稠密图更优      │ 稀疏图更优      │
│ 不需要排序      │ 需要排序边      │
└─────────────────┴─────────────────┘
```

## 五、最短路径 [⬆️ 返回目录](#toc)

### 5.1 单源最短路径 — Dijkstra算法

```
思路：贪心策略，每次选距离源点最近的未确定顶点

伪代码：
    dist[s] = 0, dist[其他] = ∞
    for i = 1 to n:
        选 dist 最小且未确定的顶点 u
        标记 u 为确定
        for v in adj(u):
            if dist[v] > dist[u] + w(u,v):
                dist[v] = dist[u] + w(u,v)
```

| 要点 | 内容 |
|------|------|
| 时间复杂度 | O(n²) 数组 / O((n+e)logn) 堆优化 |
| 适用条件 | **边权非负** ⚠️ |
| 核心思想 | **贪心** + 松弛操作 |
| 不能处理 | 负权边 |

> 📎 **[点击打开：Dijkstra算法专题](dijkstra.md)**

### 5.2 多源最短路径 — Floyd算法

```
思路：动态规划，逐个顶点作为中转点

状态：dp[k][i][j] = 只经过 {v1...vk} 中顶点，i到j的最短距离
转移：dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])

代码（空间优化后）：
    for k in 1..n:
        for i in 1..n:
            for j in 1..n:
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
```

| 要点 | 内容 |
|------|------|
| 时间复杂度 | O(n³) |
| 空间复杂度 | O(n²) |
| 适用条件 | 可处理负权边，但不能有负权回路 |
| 核心思想 | **动态规划** |
| 额外功能 | 可求传递闭包 |

> 📎 **[点击打开：Floyd算法专题](floyd.md)**

### 5.3 最短路径算法对比

```
┌─────────────────┬─────────────────┬─────────────────┐
│    Dijkstra     │     Floyd       │   BFS(无权图)   │
├─────────────────┼─────────────────┼─────────────────┤
│ 单源            │ 多源            │ 单源(无权)      │
│ 贪心            │ 动态规划        │ BFS             │
│ O(n²)           │ O(n³)           │ O(n+e)          │
│ 边权≥0          │ 可有负权        │ 无权图          │
│ 不能有负权边    │ 不能有负权回路  │ 只能无权        │
└─────────────────┴─────────────────┴─────────────────┘
```

## 六、拓扑排序 [⬆️ 返回目录](#toc)

### 6.1 基本概念

- **AOV网**：用顶点表示活动，边表示优先关系的有向无环图
- **拓扑排序**：将AOV网中所有顶点排成线性序列，保持优先关系

### 6.2 Kahn算法（入度法）

```
思路：不断删除入度为0的顶点

伪代码：
    计算所有顶点入度
    将所有入度为0的顶点入队
    while !Queue.empty():
        u = Queue.pop()
        输出 u
        for v in adj(u):
            indegree[v]--
            if indegree[v] == 0:
                Queue.push(v)
    if 输出顶点数 < n: 存在环
```

### 6.3 DFS法

```
思路：DFS后序遍历的逆序

伪代码：
    DFS(v):
        visited[v] = true
        for w in adj(v):
            if !visited[w]:
                DFS(w)
        Stack.push(v)  // 后序入栈

    // 最终栈的弹出顺序即为拓扑序
```

| 方法 | 时间复杂度 | 特点 |
|------|-----------|------|
| Kahn | O(n+e) | 直观，可判环 |
| DFS | O(n+e) | 利用后序遍历 |

## 七、关键路径 [⬆️ 返回目录](#toc)

### 7.1 基本概念

- **AOE网**：用边表示活动的带权有向无环图
- **关键路径**：从源点到汇点的最长路径（决定最短工期）
- **关键活动**：关键路径上的活动，延迟会影响总工期

### 7.2 求解步骤

```
Step 1: 拓扑排序，确定事件顺序
Step 2: 求 ve（最早发生时间）—— 正向递推
        ve[j] = max{ve[i] + w(i,j)}  (i→j)

Step 3: 求 vl（最迟发生时间）—— 反向递推
        vl[i] = min{vl[j] - w(i,j)}  (i→j)

Step 4: 求活动的时间
        e(i) = ve[k]      (活动ai从vk出发)
        l(i) = vl[j] - w  (活动ai到vj结束)

Step 5: 关键活动：e(i) == l(i) 的活动
```

### 7.3 关键公式速记

```
┌─────────────────────────────────────────────┐
│  ve(源点) = 0                               │
│  ve(j) = max{ve(i) + w(i,j)}  正向拓扑序    │
│                                             │
│  vl(汇点) = ve(汇点)                        │
│  vl(i) = min{vl[j] - w(i,j)}  逆向拓扑序    │
│                                             │
│  关键活动：e == l                            │
│  关键路径：由关键活动构成的路径              │
└─────────────────────────────────────────────┘
```
